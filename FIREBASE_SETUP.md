# Firebase Setup Guide

Complete step-by-step guide to set up Firebase for the AI Finance Platform.

## Prerequisites
- Google account
- Flutter project created
- `flutterfire_cli` installed

## Step 1: Create Firebase Project

### 1.1 Create Project
1. Visit [Firebase Console](https://console.firebase.google.com)
2. Click "Create project"
3. Enter project name: `AI Finance Platform`
4. Accept terms and click "Continue"
5. Disable Google Analytics (optional)
6. Click "Create project"

### 1.2 Wait for Project Creation
- Firebase will set up your project
- Takes 1-2 minutes

## Step 2: Configure FlutterFire

### 2.1 Install FlutterFire CLI
```bash
dart pub global activate flutterfire_cli
```

### 2.2 Configure Your Project
Navigate to your project directory and run:

```bash
flutterfire configure
```

This will:
- Prompt you to select platforms (Android, iOS, Web)
- Create configuration automatically
- Generate `firebase_options.dart`

## Step 3: Enable Authentication

### 3.1 Go to Authentication
In Firebase Console:
1. Select your project
2. Go to **Authentication**
3. Click **Get Started**

### 3.2 Enable Email/Password
1. Click **Sign-in method** tab
2. Click **Email/Password**
3. Toggle **Enable**
4. Click **Save**

### 3.3 Create Test User (Optional)
1. Go to **Users** tab
2. Click **Add user**
3. Enter email and password
4. Click **Add user**

## Step 4: Set Up Firestore Database

### 4.1 Create Database
1. Go to **Firestore Database**
2. Click **Create database**
3. Select region (closest to users)
4. Choose **Start in test mode**
5. Click **Create**

### 4.2 Configure Security Rules

Replace default rules with:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users collection
    match /users/{uid} {
      allow read, write: if request.auth.uid == uid;
    }
    
    // Finance data collection
    match /finance_data/{document=**} {
      allow read, write: if request.auth.uid == resource.data.uid;
      allow create: if request.auth.uid == request.resource.data.uid;
    }
    
    // Recommendations collection
    match /recommendations/{document=**} {
      allow read, write: if request.auth.uid == resource.data.uid;
      allow create: if request.auth.uid == request.resource.data.uid;
    }
    
    // Notifications collection
    match /notifications/{document=**} {
      allow read, write: if request.auth.uid == resource.data.uid;
      allow create: if request.auth.uid == request.resource.data.uid;
    }
    
    // Investment history collection
    match /investment_history/{document=**} {
      allow read, write: if request.auth.uid == resource.data.uid;
      allow create: if request.auth.uid == request.resource.data.uid;
    }
  }
}
```

### 4.3 Publish Rules
1. Click **Publish**
2. Confirm changes

## Step 5: Create Firestore Collections

### 5.1 Create Collections Structure

Navigate to Firestore and create these collections:

#### Users Collection
```
Collection: users
Documents: (auto-generated by UID)
Fields:
- uid: string
- email: string
- name: string
- profileImageUrl: string (optional)
- createdAt: timestamp
- updatedAt: timestamp
```

#### Finance Data Collection
```
Collection: finance_data
Fields:
- id: string
- uid: string
- income: number
- expenses: number
- age: number
- goalAmount: number
- months: number
- riskLevel: string
- createdAt: timestamp
- updatedAt: timestamp
```

#### Recommendations Collection
```
Collection: recommendations
Fields:
- id: string
- uid: string
- recommendation: string
- category: string
- createdAt: timestamp
```

#### Notifications Collection
```
Collection: notifications
Fields:
- id: string
- uid: string
- title: string
- body: string
- type: string
- createdAt: timestamp
- isRead: boolean
```

## Step 6: Enable Cloud Messaging (Optional)

### 6.1 Go to Cloud Messaging
1. In Firebase Console, go to **Cloud Messaging**
2. Note your **Server API Key** and **Sender ID**

### 6.2 Configure App
1. In `notification_service.dart`, update if needed
2. Test with Android/iOS push notifications

## Step 7: Firebase Storage Setup (Optional)

### 7.1 Enable Storage
1. Go to **Storage**
2. Click **Get Started**
3. Choose production or test mode
4. Click **Done**

### 7.2 Configure Rules
```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /users/{uid}/{allPaths=**} {
      allow read, write: if request.auth.uid == uid;
    }
  }
}
```

## Step 8: Update Environment Variables

### 8.1 Create `.env` file
```env
# OpenAI API
OPENAI_API_KEY=your_key_here

# Alpha Vantage API
ALPHA_VANTAGE_API_KEY=your_key_here
```

### 8.2 Verify Firebase Options
Check `lib/firebase_options.dart`:
```dart
static const web = FirebaseOptions(
  apiKey: '...',
  appId: '...',
  messagingSenderId: '...',
  projectId: 'your-project-id',
);
```

## Step 9: Test Connection

### 9.1 Run App
```bash
flutter run
```

### 9.2 Test Authentication
1. Navigate to Sign Up screen
2. Create new account
3. Check Firebase Console → Authentication
4. Should see new user

### 9.3 Test Firestore
1. Navigate to Finance Form
2. Fill and submit form
3. Check Firebase Console → Firestore
4. Should see new document in `finance_data` collection

## Troubleshooting

### Issue: Firebase app not initialized
**Solution:**
```bash
flutter pub get
flutter clean
flutter run
```

### Issue: Firestore rules denied
**Solution:**
- Verify `uid` in request matches authenticated user
- Check security rules are correct
- Ensure user is authenticated

### Issue: Config files not found
**Solution:**
```bash
flutterfire configure --overwrite
```

### Issue: Android build failure
**Solution:**
- Update Google Services plugin
- Check `android/build.gradle`
- Run `flutter clean` and rebuild

## Security Best Practices

1. **Never commit API keys** - Use environment variables
2. **Validate all inputs** - Server and client-side
3. **Use HTTPS** - Always
4. **Keep rules updated** - Review regularly
5. **Monitor usage** - Check Firebase quotas
6. **Backup data** - Regular exports
7. **Audit logs** - Enable and review
